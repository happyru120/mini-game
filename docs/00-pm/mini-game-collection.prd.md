## 1. 프로젝트 배경 및 개요

### 1.1 왜 지금인가

현대인의 스트레스 지수는 매년 높아지고 있다. 직장에서 치이고, 학교에서 치이고, 집에 돌아와서도 완전히 해소되지 않는 감정들이 쌓인다.

기존 스트레스 해소 방법의 문제:

- **명상/마음챙김 앱**: 효과는 있지만 지금 당장 열 받을 때는 쓸 수 없다
- **일반 게임**: 배워야 할 것이 많고, 져서 더 화나는 경우가 생긴다
- **운동**: 몸을 일으켜야 하는 진입 장벽이 있다
- **SNS 스크롤**: 오히려 비교와 자극으로 스트레스가 가중된다

파괴형 게임의 카타르시스 효과는 심리학적으로 검증된 개념이다. 뭔가를 "부수는" 행위는 즉각적인 감정 해방감을 준다. 하지만 이를 간단하게, 즉시, 재미있게 제공하는 서비스는 없었다.

### 1.2 프로젝트 목표

**비전**: "열 받은 사람이 브라우저 주소창을 열었을 때 가장 먼저 생각나는 곳"

**OKR (3개월 목표)**

| Objective | Key Result |
|-----------|-----------|
| 즉각적인 스트레스 해소 경험 제공 | 평균 세션 시간 3분 이상 |
| 입소문을 타는 바이럴 콘텐츠 | 카카오톡/Discord 공유 수 월 1,000회 이상 |
| 재방문을 유도하는 경험 | 7일 재방문율 30% 이상 |
| 게임 컬렉션 완성 | Phase 1: 핵심 게임 4종 출시 |

## 3. 핵심 가치 제안

### 3.1 JTBD (Jobs To Be Done) 분석

**기능적 JOB**: 스트레스를 받는 상황에서 즉각적으로 감정을 해소하고 싶다

**감정적 JOB**: "나 지금 진짜 열 받았어" 라는 감정을 안전하게, 유쾌하게 표현하고 싶다

**사회적 JOB**: "이거 진짜 웃겨서 기분 풀렸어"를 친구에게 공유하며 연대감을 형성하고 싶다

### 3.2 가치 제안 캔버스

**해소하는 고통 (Pain Relievers)**
- 진입 장벽 제로: 설치 없이, 로그인 없이, 3초 안에 시작
- 실패해도 괜찮은 구조: 지는 게임이 아니라, 부수는 게임
- 짧은 세션: 5분 이상 투자 필요 없음

**만들어주는 이득 (Gain Creators)**
- 즉각적 카타르시스: 깨지고, 터지고, 부서지는 시각/청각 피드백
- "오늘의 분노 수치" 점수화: 내 감정이 숫자로 인정받는 경험
- 유쾌한 멘트: "오늘도 고생했다", "진짜 열 받았구나!"
- 공유 가능성: 내 분노 점수를 친구와 비교

### 카테고리 A: 🔥 스트레스 해소 게임 (6종)

#### 공통 원칙

- **세션 길이**: 1-5분
- **실패 없음**: 지는 구조가 아니라 "얼마나 시원하게 부쉈는가"를 측정
- **즉각 피드백**: 모든 파괴 행위에 사운드 + 시각 이펙트 필수
- **분노 수치**: 각 게임은 "분노 게이지" 또는 "카타르시스 점수" 산출

### Game 01: 접시 날려라 (Dish Smash)

**한 줄 설명**: 날아오는 접시를 클릭/터치해서 산산조각 낸다

| 항목 | 내용 |
|------|------|
| **핵심 메카닉** | 화면을 가로질러 날아오는 접시(또는 컵, 꽃병 등)를 클릭/탭 |
| **조작법** | 마우스 클릭 / 터치 탭 |
| **세션 시간** | 60초 타임어택 |
| **난이도 곡선** | 시간이 지날수록 접시 속도 증가, 여러 개 동시 등장 |
| **파괴 이펙트** | 파편이 사방으로 튀는 애니메이션 + 유리 깨지는 소리 |
| **점수 시스템** | 깬 접시 수 × 크기 배율 = "오늘의 분노 지수" |
| **카피** | "날아오면 다 깨버려!" / "산산조각 나는 기분 어때?" |
| **기술** | Canvas 2D, requestAnimationFrame, 파티클 시스템 |

**접시 종류 (Progressive Unlock)**
1. 일반 접시 (기본)
2. 찻잔 (작고 빠름, 점수 2배)
3. 꽃병 (크고 느림, 파편 많음)
4. "상사 얼굴 접시" (이스터에그 - 분노 게이지 MAX 시 등장)

### Game 02: 분노의 타이핑 (Rage Type)

**한 줄 설명**: 화면에 나타나는 텍스트를 미친 듯이 빠르게 쳐서 점수를 올린다

| 항목 | 내용 |
|------|------|
| **핵심 메카닉** | 화면에 단어/문장이 등장, 빠르게 타이핑할수록 폭발 이펙트 |
| **조작법** | 키보드 타이핑 (모바일: 텍스트 입력창) |
| **세션 시간** | 90초 |
| **텍스트 종류** | "열받아!", "왜 나한테 이래!", "퇴직하고 싶다", "아 진짜!" 등 공감 텍스트 |
| **파괴 이펙트** | 정확히 칠수록 글자가 폭발 + 화면 흔들림 (빠를수록 강하게) |
| **점수 시스템** | WPM × 정확도 × 연속 타격 콤보 = 분노 타이핑 지수 |
| **카피** | "손가락으로 다 토해내!" / "빠를수록 시원해!" |
| **특이 기능** | "자유 모드": 아무 키나 눌러도 이펙트 발생 (글자 맞춤 없이 그냥 두드리기) |

**텍스트 카테고리**
- 직장인 분노 팩: "야근하기 싫다", "그 회의는 왜 한 거야"
- 수험생 분노 팩: "왜 틀렸어 나", "공부 싫다"
- 범용 팩: "아!!!!", "왜!!!!", "진짜!!!!"

### Game 03: 스트레스볼 연타 (Stress Ball Smash)

**한 줄 설명**: 화면 가득 채워진 스트레스볼을 미친 듯이 클릭/터치해서 터뜨린다

| 항목 | 내용 |
|------|------|
| **핵심 메카닉** | 스트레스볼을 클릭하면 변형되다가 터짐. 빠를수록 연쇄 폭발 |
| **조작법** | 마우스 클릭 (연타) / 멀티터치 탭 |
| **세션 시간** | 무제한 (피로해서 그만두는 자연 종료) |
| **콤보 시스템** | 0.5초 이내 연속 클릭 시 콤보 배율 증가 (MAX 10배) |
| **파괴 이펙트** | 공이 납작해지다가 "팡!" 하고 터지는 애니메이션 + 인정 멘트 |
| **점수 시스템** | 터뜨린 수 × 콤보 배율 = 연타 카타르시스 지수 |
| **카피** | "터질 때까지 눌러!" / "콤보 끊지 마!" |
| **공 종류** | 기본 스트레스볼 → 행운의 황금볼 → 분노의 빨간볼 (배율 3배) |

### Game 04: 버블랩 터뜨리기 (Bubble Wrap Pop)

**한 줄 설명**: 무한 버블랩을 손가락으로 터뜨리는 순수한 감촉 쾌감 게임

| 항목 | 내용 |
|------|------|
| **핵심 메카닉** | 그리드 형태의 버블랩. 하나씩 또는 드래그로 쓸어서 터뜨리기 |
| **조작법** | 클릭/터치. 드래그로 여러 개 한번에 터뜨리기 가능 |
| **세션 시간** | 무제한 (모두 터뜨리면 새 시트 자동 등장) |
| **특이 기능** | "정신없이 흔들기 모드": 디바이스 흔들기(모바일 자이로) 또는 스페이스바 연타로 랜덤 폭발 |
| **파괴 이펙트** | 각 버블마다 "뻥!" 소리 + 작은 파티클. 전체 완료 시 축하 이펙트 |
| **점수 시스템** | 없음 (순수 힐링 모드). "오늘 터뜨린 버블: 247개" |
| **카피** | "생각 없이 그냥 터뜨려" / "중독주의" |
| **특이 버블** | 황금 버블 (터뜨리면 특별 소리), 슈퍼 버블 (주변 5개 연쇄 폭발) |

### Game 05: 종이 찢기 (Paper Rip)

**한 줄 설명**: 성가신 서류, 이메일, 고지서를 손으로 찢어버리는 시뮬레이션

| 항목 | 내용 |
|------|------|
| **핵심 메카닉** | 마우스 드래그 또는 터치 스와이프로 종이를 찢기. 방향과 속도에 따라 다른 찢기 패턴 |
| **조작법** | 클릭+드래그 / 터치 스와이프 |
| **세션 시간** | 1분 (종이 여러 장 등장) |
| **종이 종류** | "야근 보고서", "청구서", "말도 안 되는 지시문", "불합격 통보" (패러디 텍스트) |
| **파괴 이펙트** | 종이가 실제처럼 찢어지는 물리 시뮬레이션 + 찢는 소리 |
| **점수 시스템** | 찢어진 조각 수 × 속도 보너스 = "해방감 지수" |

### 5.2 핵심 게임 루프

```javascript
// GameLoop.js - 공통 게임 루프 베이스
class GameLoop {
  constructor(updateFn, renderFn) {
    this.update = updateFn;
    this.render = renderFn;
    this.lastTime = 0;
    this.targetFPS = 60;
    this.frameTime = 1000 / this.targetFPS;
  }

  start() {
    requestAnimationFrame((time) => this.loop(time));
  }

  loop(timestamp) {
    const deltaTime = timestamp - this.lastTime;
    if (deltaTime >= this.frameTime) {
      this.update(deltaTime);
      this.render();
      this.lastTime = timestamp;
    }
    requestAnimationFrame((time) => this.loop(time));
  }
}
```

### 5.4 입력 통합 시스템

```javascript
// InputManager.js - 터치/마우스/키보드 통합
class InputManager {
  constructor(canvas) {
    this.canvas = canvas;
    this.events = new EventEmitter();
    this._bindMouse();
    this._bindTouch();
    this._bindKeyboard();
  }

  _normalizePosition(e) {
    const rect = this.canvas.getBoundingClientRect();
    return {
      x: (e.clientX - rect.left) * (this.canvas.width / rect.width),
      y: (e.clientY - rect.top) * (this.canvas.height / rect.height),
    };
  }
  // 터치 이벤트는 동일한 인터페이스로 변환하여 emit
}
```

### 6.2 피드백 시스템 계층

모든 파괴 행위에는 3단계 피드백이 동시에 발생해야 한다.

```
[ 행위 발생 ]
     |
     +-- 즉각 시각 피드백 (0ms): 오브젝트 변형/파괴 애니메이션
     |
     +-- 즉각 청각 피드백 (0ms): 파괴음 재생
     |
     +-- 점수/콤보 피드백 (100ms): 팝업 숫자, 콤보 텍스트

     |
     +-- 달성 피드백 (선택적): 마일스톤 도달 시 특별 이펙트
```

### 7.2 Web Audio API 아키텍처

```javascript
// SoundManager.js
class SoundManager {
  constructor() {
    this.ctx = new AudioContext();
    this.masterGain = this.ctx.createGain();
    this.masterGain.connect(this.ctx.destination);
    this.sounds = {};
  }

  async loadSound(name, url) {
    const response = await fetch(url);
    const buffer = await response.arrayBuffer();
    this.sounds[name] = await this.ctx.decodeAudioData(buffer);
  }

  play(name, options = {}) {
    const source = this.ctx.createBufferSource();
    source.buffer = this.sounds[name];
    source.playbackRate.value = options.pitch || 1;
    source.connect(this.masterGain);
    source.start(0);
  }

  // 볼륨 0 유지 후 user gesture 이후 resume (autoplay policy 대응)
  resume() { this.ctx.resume(); }
}
```

### 7.3 프로시저럴 사운드 (에셋 없이 구현)

초기 버전은 외부 사운드 파일 없이 Web Audio API Oscillator로 직접 생성:

- **버블 팝**: 짧은 Sine 파형 → 빠른 Decay
- **타이핑**: 짧은 Sawtooth → 즉각 Gain 0
- **폭발**: White Noise → Low-pass Filter → Exponential Decay

### 8.1 분노 지수 공식

각 게임의 점수는 "분노 지수"로 통일 표시. 실제 계산식은 게임마다 다르지만 결과값 스케일은 통일.

| 게임 | 기본 공식 | 최대 점수 |
|------|-----------|-----------|
| 접시 날려라 | `(깬 수 × 크기 배율 × 콤보) × 100` | 99,999 |
| 분노의 타이핑 | `WPM × 정확도(%) × 콤보` | 99,999 |
| 스트레스볼 연타 | `터뜨린 수 × 연타 속도 보너스` | 99,999 |
| 버블랩 터뜨리기 | `없음 (힐링 모드)` | - |
| 종이 찢기 | `조각 수 × 속도 보너스 × 종이 종류 배율` | 99,999 |
| 상사한테 던지기 | `명중 정확도 × 물체 무게 × 거리 보너스` | 99,999 |

| US-10 | 유저로서, 내 역대 최고 분노 지수를 확인하고 싶다. 그래서 오늘이 얼마나 힘든 날이었는지 돌아볼 수 있다. | 낮음 | Given 결과 화면, When "기록 보기" 탭, Then 최근 7일 분노 지수 그래프 표시 |

## 13. Pre-mortem 리스크 분석

12개월 후 이 프로젝트가 실패했다면 무슨 일이 일어난 것인가?

| # | 실패 시나리오 | 카테고리 | 가능성 | 영향 | 예방 전략 |
|---|-------------|---------|:------:|:----:|-----------|
| R1 | 파괴 이펙트가 기대에 못 미쳐 "시원함"을 주지 못함. 유저가 한 번 해보고 다시 오지 않음 | 가치 | 높음 | 높음 | Phase 1 프로토타입에서 5명 이상 유저 테스트, "시원했나요?" 즉각 피드백 수집 |
| R2 | 모바일에서 터치 반응이 너무 느려 감동이 반감됨 | 사용성 | 중간 | 높음 | 입력→이펙트 지연 50ms 목표, 저사양 기기(Android 보급형) 필수 테스트 |
| R3 | 60fps 미달성으로 파티클 이펙트가 뚝뚝 끊김 | 실행 가능성 | 중간 | 높음 | 오브젝트 풀링 필수, 파티클 수 기기 성능 자동 조절 구현 |
| R4 | 게임이 너무 단순해서 1분도 안 되어 질림 | 가치 | 중간 | 중간 | 난이도 곡선 + 콤보 시스템으로 60초 이상 몰입 설계 |
| R5 | 공유 카드가 매력 없어 바이럴이 일어나지 않음 | 시장 | 중간 | 높음 | 공유 이미지 A/B 테스트, 멘트 카피라이팅에 집중 |
| R6 | 사운드가 없거나 빈약해서 파괴 쾌감이 크게 줄어듦 | 사용성 | 낮음 | 높음 | 프로시저럴 사운드로 초기부터 적용, 사운드 없는 버전과 비교 테스트 |
| R7 | "직장 폭력" 또는 "괴롭힘 조장" 논란 발생 | 시장/팀 | 낮음 | 높음 | 게임 시작 전 "상상 속 이야기" 명시, 타겟을 익명 실루엣으로 처리, 폭력성 없는 유쾌한 톤 유지 |
| R8 | 브라우저 Autoplay Policy로 사운드가 안 나와 초기 인상이 나쁨 | 실행 가능성 | 높음 | 중간 | 첫 클릭 이후 AudioContext.resume() 호출, "사운드 켜기" 명시적 버튼 제공 |

**출시 전 반드시 해결해야 할 Top 3 리스크**:
1. **R1 (파괴 이펙트 불만족)**: 파티클 수, 사운드 볼륨, 화면 흔들림 강도를 유저 테스트로 검증
2. **R2 (모바일 터치 지연)**: 입력 지연 50ms 이하 달성 후 출시
3. **R8 (사운드 자동재생 차단)**: 명시적 사운드 활성화 UX 구현 필수
